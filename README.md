# Fitness Platform

一个可本地运行的健身软件原型，目标是覆盖市面常见健身 App 的核心链路：

- 用户画像与目标管理
- 基于国际标准的训练方案生成
- 饮食与营养计划
- 训练日志/打卡
- 数据分析仪表盘
- 社区动态与挑战

## 技术栈

- React 19 + TypeScript
- Vite
- 本地 `localStorage` 持久化（无后端依赖，便于快速验收）

## Android 原生版本

仓库内新增了原生 Android 子工程：

- `android-app/`

能力覆盖：

- 本地注册与登录
- 健身计划生成
- 训练记录与最近 7 天分析
- 饮食记录与建议
- 目标设定与进度监控
- `Master Agent` + 多个 `SubAgent` 的 5 分钟巡检/重启机制

构建方式：

```bash
cd android-app
./gradlew.bat assembleDebug
```

## 运行方式

```bash
npm install
npm run dev
```

打开浏览器访问终端里显示的本地地址（默认通常是 `http://localhost:5173`）。

生产构建：

```bash
npm run build
```

## 主要功能说明

### 1) 用户画像与目标列表

- 编辑年龄、身高体重、目标类型、经验等级、器械条件、训练频率、恢复能力、睡眠等。
- 目标列表支持新增目标（体重、体脂、力量、距离、一致性）并显示进度条。

### 2) 训练方案生成（国际标准驱动）

- 根据画像生成周训练拆分、每肌群周训练量、每周有氧分钟数、每日动作与强度建议。
- 可点击 `Regenerate Plans` 重新生成训练和饮食计划。

### 3) 饮食计划

- 输出每日总热量和三大营养素（蛋白/碳水/脂肪）。
- 提供分餐建议、饮水目标、营养执行备注。

### 4) 运动记录

- 可新增训练日志（日期、训练类型、时长、RPE、消耗、备注）。
- 支持已完成/待完成状态切换。

### 5) 数据分析

- 最近 7 天训练时长趋势图。
- 完成率、平均 RPE、目标平均进度等指标。

### 6) 社交与挑战

- 挑战列表（加入/退出）。
- 社区发帖、帖子搜索过滤、点赞。

## 使用的国际标准（在逻辑中落地，不是纯文案）

实现文件：`src/planner.ts`

- WHO Physical Activity Guideline：每周有氧推荐分钟数（150-300 分钟中高等活动区间的映射）。
- ACSM Resistance Training Guidance：每周训练频次与肌群训练量的结构化安排，含渐进超负荷。
- NSCA Volume/Intensity Guidance：按训练水平和恢复能力动态调整训练量。
- NASM 恢复与周期化思路：加入恢复与减量周规则。
- USDA/运动营养通用原则：按体重、目标和训练负荷推导热量与宏量营养素。

## Mock / 简化说明

- 当前为前端单体原型，数据使用本地状态 + `localStorage`，没有接入真实账号系统或后端数据库。
- 社交数据、挑战数据、部分日志为种子 mock 数据。
- 目标进度算法为演示版（可继续替换为真实追踪指标计算）。
- 饮食推荐是规则引擎基础版，不替代医疗建议。
